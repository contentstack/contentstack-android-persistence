# Skills – Contentstack Android Persistence

Source of truth for detailed guidance. Read [AGENTS.md](../AGENTS.md) first, then open the skill that matches your task.

## When to use which skill

| Skill folder | Use when |
| --- | --- |
| `dev-workflow` | Branching, CI expectations, day-to-day Gradle build/test/lint, or release/publish context. |
| `android-persistence-sdk` | Changing or documenting `RealmStore`, `SyncManager`, `SyncStore`, `SyncPersistable`, or how they integrate with Contentstack `Stack` / sync. |
| `java-android` | Java style, packages, or where source and resources live under `app/src/main/`. |
| `android-platform` | Gradle/AGP/Realm versions, `gradle.properties` coordinates, dependency bumps, or Maven publishing plugin behavior. |
| `testing` | Adding unit or instrumented tests, fixtures, or policies around secrets in tests. |
| `code-review` | Preparing or reviewing a PR against team expectations. |

Each folder contains `SKILL.md` with YAML frontmatter (`name`, `description`).
